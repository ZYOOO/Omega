# Requirement 模板

复制到新建需求的描述中使用。

## 背景

说明当前问题、用户场景或为什么要做这件事。

## 目标

一句话描述完成后用户能得到什么。

## 范围

- 本次要做什么。
- 预计会影响哪些页面、接口、数据或运行时行为。

## 非范围

- 本次明确不做什么。
- 哪些想法留到后续。

## 验收标准

- [ ] 可以由用户或测试明确判断的完成条件。
- [ ] 页面、接口或命令的预期结果。
- [ ] 需要兼容 light / dark、桌面 / 移动端或 Electron 的地方。
